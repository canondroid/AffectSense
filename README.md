# AffectSense — Real-Time Cognitive Stress Detection

A macOS desktop application that detects cognitive stress in real time using webcam-based facial analysis, keystroke timing, and active application context. When elevated stress is detected, it surfaces a non-intrusive notification with a plain-language explanation and lets you correct the model — which retrains itself in the background.

Built for Apple Silicon macOS. Designed for Human-Centered AI research.

## 3. Installation

### Step 1 — Clone the repository

```bash
git clone <repo-url>
cd stressAware
```

### Step 2 — Install Python 3.11

MediaPipe requires Python 3.11. Check what you have:

```bash
python3 --version
```

If you don't have 3.11, install it via Homebrew:

```bash
brew install python@3.11
```

### Step 3 — Create a virtual environment

```bash
/opt/homebrew/bin/python3.11 -m venv .venv
source .venv/bin/activate
```

> **Important:** Always activate the venv before running any command in this project. All subsequent steps assume the venv is active.

### Step 4 — Install libomp (required by XGBoost on macOS)

```bash
brew install libomp
```

### Step 5 — Install Python dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

This installs PyQt6, MediaPipe, XGBoost, PyTorch (with MPS support), SHAP, pynput, pyobjc, and supporting libraries. Expect it to take a few minutes.

> **Note:** The requirements file uses `opencv-contrib-python`. Do **not** separately install `opencv-python` — they conflict. MediaPipe will install the correct version automatically.

### Step 6 — Verify the installation

```bash
python -c "import mediapipe, xgboost, torch, PyQt6; print('All dependencies OK')"
python -c "import torch; print('MPS available:', torch.backends.mps.is_available())"
```

## 4. macOS Permissions

The app requires two macOS privacy permissions. Grant both before running anything that opens the webcam or keyboard monitor.

### Camera

1. Open **System Settings → Privacy & Security → Camera**
2. Enable access for **Terminal** (or whichever terminal app you use)

### Input Monitoring (Keystroke Timing)

This is the trickiest permission on macOS 15. The permission must be granted to the **exact Python binary inside your `.venv`**, not the system Python.

1. Find your venv Python path:
   ```bash
   which python
   # Should print something like: /path/to/stressAware/.venv/bin/python
   ```
2. Open **System Settings → Privacy & Security → Input Monitoring**
3. Click **+** and navigate to `.venv/bin/python` inside the project folder
4. Enable the toggle next to it
5. Restart your terminal

> **macOS 15 gotcha:** If you see "Checking Input Monitoring permission — please press any key..." at startup and nothing happens after pressing a key, you have the wrong binary whitelisted. Remove any existing entries and re-add the exact path shown by `which python`.

### Verify permissions

```bash
python -m src.utils.permissions_check
```

If both permissions are working, this prints `All permission checks passed.`

## 5. Training the Stress Classifier

The XGBoost stress classifier is entirely personal — it learns from **your own behaviour**. You must collect your own labelled sessions before training.

### Step 1 — Collect labelled sessions

Run the session collector while actually doing work. The collector runs all sensors (webcam, keyboard, app monitor) for 20 minutes and writes one feature row every 30 seconds.

**Collect calm sessions** (do relaxed, low-pressure work):

```bash
python -m src.training.collect_session --label calm
```

**Collect stress sessions** (do something that genuinely stresses you — tight deadline, difficult problem, time pressure):

```bash
python -m src.training.collect_session --label stress
```

**Recommended minimum:** 4 calm sessions + 4 stress sessions (≥ 160 data points total). More sessions = better model. You can collect sessions over multiple days.

**What gets saved:** Each session writes a CSV to `data/sessions/labelled/` with a name like `session_calm_20260420_143022.csv`. The file contains 28 feature columns and a `label` column (0 = calm, 1 = stress). No raw video or keystrokes are stored — only derived numerical features.

**During collection you will see a live status line like:**

```
  [collecting]  12.4/20 min | windows: 24 | blink_rate: 14.2/min | wpm: 62 | remaining: 7.6 min
```

The first 60 seconds show `[warming up]` while the sliding window fills. This is normal.

Press `Ctrl+C` at any time to end a session early (partial data is saved).

### Step 2 — Train the model

Once you have at least 4 sessions (2 calm + 2 stress):

```bash
python -m src.training.train_xgb
```

This will:
- Load all CSVs from `data/sessions/labelled/`
- Apply SMOTE oversampling if either class has fewer than 50 samples
- Train an XGBoost binary classifier with early stopping
- Print accuracy, F1 score, ROC-AUC, and a confusion matrix
- Save the following files automatically:

| File | Location | Purpose |
|---|---|---|
| `xgb_stress_v1.json` | `models/xgb_model/` | Trained XGBoost model |
| `feature_scaler.pkl` | `models/xgb_model/` | StandardScaler for feature normalisation |
| `model_meta.json` | `models/xgb_model/` | Training stats (F1, sample counts, timestamp) |
| `version.txt` | `models/xgb_model/` | Version counter for model rotation |
| `explainer.pkl` | `models/shap/` | SHAP TreeExplainer for feature attribution |

You do not need to move any files — the paths are resolved automatically.

**Example output:**

```
AffectSense — XGBoost Training
----------------------------------------
  Loaded: session_calm_20260420_143022.csv (39 rows)
  Loaded: session_stress_20260420_161045.csv (38 rows)
  Total session rows: 77
  Class distribution: calm=39, stress=38
  Train: 61 | Val: 16

  Accuracy:  0.8750
  F1:        0.8824
  ROC-AUC:   0.9375

Training complete. F1=0.8824
```

### Step 3 — Retrain after corrections (automatic)

Once the app is running, every time you press "That's wrong" on a notification, the correction is logged to `data/sessions/corrections/corrections.jsonl`. After **10 corrections**, the model automatically retrains in the background and hot-reloads — no restart required.

You can also trigger manual retraining from the dashboard (Model Status tab → "Retrain now").

## 6. Optional: FER Model (Facial Expression)

The FER (Facial Expression Recognition) model improves detection by analysing facial expressions in real time. It is **optional** — the app works without it using a neutral 0.5 probability fallback for the `fer_stress_prob` feature.

The FER model uses EfficientNet-B0 fine-tuned on the RAF-DB dataset.

### Step 1 — Download RAF-DB

1. Go to: http://www.whdeng.cn/RAF/model1.html
2. Fill in the academic access request form
3. Download `rafdb_basic.tar.gz`
4. Extract it so the directory structure looks exactly like this:

```
data/raw/RAF-DB/basic/
├── Image/
│   └── aligned/
│       ├── train_00001_aligned.jpg
│       ├── train_00002_aligned.jpg
│       └── ...
└── EmoLabel/
    └── list_patition_label.txt
```

### Step 2 — Preprocess the dataset

```bash
python -m src.training.prepare_rafdb
```

This validates all image paths and writes two CSV index files to `data/processed/`. It does **not** load images into memory.

### Step 3 — Train the FER model

```bash
python -m src.training.train_fer
```

Training runs in two phases:
- **Phase 1 (3 epochs):** Backbone frozen, only the classification head trains. Fast (~5 min on M2).
- **Phase 2 (up to 10 epochs):** Full model unfreezes, trains with cosine annealing and early stopping. (~30–60 min on M2 MPS).

The best checkpoint (by validation accuracy) is saved to:

```
models/fer_model/efficientnet_rafdb.pt
```

The app automatically detects and loads this file on startup if it exists.

## 7. Running the Application

### Prerequisites checklist

- [ ] Venv is active (`source .venv/bin/activate`)
- [ ] Camera permission granted to Terminal
- [ ] Input Monitoring permission granted to `.venv/bin/python`
- [ ] At least one `train_xgb` run completed (model files exist in `models/xgb_model/`)

### Launch

```bash
python -m src.ui.main_app
```

The app starts as a **menu bar / system tray application** — no dock icon appears. Look for the stress indicator circle in your menu bar (top-right area of screen).

### Launch flags

| Flag | Effect |
|---|---|
| `--skip-permissions` | Skip camera and input monitoring checks (useful during development) |
| `--dashboard` | Open the dashboard window immediately on launch |
| `--debug-tier 1` | Simulate a tier-1 notification after 2 seconds (UI testing, no model needed) |
| `--debug-tier 2` | Simulate a tier-2 (Warning) notification |
| `--debug-tier 3` | Simulate a tier-3 (High Stress) notification |

```bash
# Development mode — skip permissions, open dashboard, simulate a notification
python -m src.ui.main_app --skip-permissions --dashboard --debug-tier 2
```

### What you will see

1. **System tray icon** — colour-coded circle: green (calm), amber (caution), red (high stress)
2. **Notifications** — slide in from the bottom-right after 30-second scoring cycles if stress is detected. Auto-dismiss after 45 seconds. Three buttons:
   - **See why** — opens the XAI panel showing which signals drove the score
   - **Dismiss** — logs that you agreed with the assessment
   - **That's wrong** — opens a correction form; logged correction contributes to retraining
3. **Dashboard** — right-click the tray icon → "Open Dashboard" to see today's stress timeline, 7-day history, and model status

### Tray icon menu

Right-click (or control-click) the tray icon for:
- **Open Dashboard** — shows today's metrics, history, and model info
- **Pause for 15 minutes** — suppresses all notifications; auto-resumes
- **Resume monitoring** — re-enables after a manual pause
- **Quit** — shuts down cleanly

## 8. How the App Works

### Sensing pipeline (background thread, 30 fps)

Every frame from the webcam is processed by MediaPipe FaceMesh (478 landmarks). Every 30 seconds, a feature vector of 28 values is assembled:

| Group | Features |
|---|---|
| Facial (10) | EAR mean/std, blink rate, PERCLOS, head yaw/pitch variance, brow compression, jaw tension, upper lip raiser, FER stress probability |
| Behavioural (9) | Inter-keystroke interval (mean/std/CV), backspace rate, WPM, burst count, app switch rate, time on current app |
| App context (5) | One-hot: development / browser / communication / writing / other |
| Contextual (4) | Session duration, time since last intervention, last intervention agreed, hour of day, daily stress mean |

### Scoring

The 28 features are scaled and passed to XGBoost. The output probability is scaled to 0–100:

| Score range | Tier | Meaning |
|---|---|---|
| 0–44 | 0 — Calm | No action |
| 45–64 | 1 — Caution | Mild cognitive load notification |
| 65–79 | 2 — Warning | Elevated stress, suggest short break |
| 80–100 | 3 — High Stress | Strong recommendation to break; Do Not Disturb activated |

### Cooldowns (prevent notification fatigue)

- **15 minutes** between notifications of the same tier
- **5 minutes** between any two notifications
- **10 minutes** after pressing "That's wrong"
- Tier escalation (e.g. caution → warning) bypasses same-tier cooldown

### Explainability

Every notification is backed by SHAP (SHapley Additive exPlanations) TreeExplainer values. The top 3 contributing features are shown in plain language. The "See why" panel shows all 28 feature values and their directional contributions.

## 10. Troubleshooting

### "Could not open webcam"

- Check **System Settings → Privacy & Security → Camera** — Terminal must have access
- Try `python -c "import cv2; cap = cv2.VideoCapture(0); print(cap.read()[0])"` — should print `True`

### Keyboard listener starts but captures no events

This is the macOS 15 Input Monitoring issue. The permission is granted to the wrong binary.

```bash
# Find the exact path that needs to be whitelisted
which python
```

Go to **System Settings → Privacy & Security → Input Monitoring**, remove existing entries, add the path printed above, and restart.

### `libxgboost.dylib` load failure at import

```bash
brew install libomp
```

### XGBoost uses CPU even on Apple Silicon

This is correct. XGBoost's `hist` method runs efficiently on CPU. Only the FER model (PyTorch) uses MPS.

### Training says "Only N samples — need at least 20"

Collect more sessions. Each 20-minute session produces ~38 rows (one every 30 seconds after the 60-second warm-up). Two sessions give ~76 rows, which is enough to train.

### Notifications never appear after training

Check that `models/xgb_model/xgb_stress_v1.json` exists:

```bash
ls -lh models/xgb_model/
```

If it's missing, run `python -m src.training.train_xgb` again. The app waits for the model to be ready and will not trigger notifications until `StressClassifier.is_ready` returns `True`.

### App does not appear in the menu bar

On macOS 15 with some system configurations, the tray icon can take a few seconds to appear. If it never appears, check that **System Tray** is enabled and that you're not running in a minimal window server context (e.g., via SSH).

### `ModuleNotFoundError` for any import

Make sure the venv is active:

```bash
source .venv/bin/activate
which python   # should point inside stressAware/.venv/
```
